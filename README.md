# CS-255-Portfolio
CS 255 Portfolio Reflection – DriverPass Project

Briefly summarize the DriverPass project. Who was the client? What type of system did they want you to design?
The DriverPass project focused on designing a system for a client named DriverPass, a company that wanted to improve how students prepare for their driver's license exams. The client needed a web-based system that would allow users to take practice tests, schedule driving lessons, manage appointments, and access educational materials. The goal of the system was to provide a complete learning platform that would help students successfully prepare for both the written and practical portions of the driver's license examination.

What did you do particularly well?
One aspect of this project that I did particularly well was gathering and documenting the client's requirements and translating those requirements into system design artifacts. Creating business requirements documents, use case diagrams, activity diagrams, and system design models helped me better understand how each component of the system interacts with the others. I also improved my ability to analyze user needs and organize those requirements into a structured software design process.

If you could choose one part of your work on these documents to revise, what would you pick? How would you improve it?
If I could revise one part of my work, I would improve the system design documentation by adding more detailed diagrams and expanding the descriptions of system interactions. While my original designs met the project requirements, adding additional detail to the UML diagrams and system workflows would make the system easier for developers and stakeholders to understand and implement.

How did you interpret the user’s needs and implement them into your system design? Why is it so important to consider the user’s needs when designing?
To interpret the user's needs, I carefully reviewed the client's interview and business requirements and used those requirements to guide each stage of the system design process. Considering the user's needs is essential because the success of any software system depends on how effectively it solves the user's problems and provides a positive experience. A system that fails to address user requirements, regardless of its technical quality, will ultimately not meet its intended purpose.

How do you approach designing software? What techniques or strategies would you use in the future to analyze and design a system?
When approaching software design, I focus on understanding the business problem first before developing technical solutions. Throughout this project, I used techniques such as requirements analysis, UML modeling, process mapping, and system architecture design to organize and communicate ideas. In future projects, I plan to continue using these strategies while placing additional emphasis on iterative design, stakeholder feedback, and user-centered development practices to create more effective and reliable software systems.
